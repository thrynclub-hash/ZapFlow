# Phase 1 Plan 01: Provisionar projeto Supabase isolado (schema + extensões + cron + edge functions)

## Objective
- **What:** Criar um projeto Supabase novo e totalmente independente (`zapflow-fase2`), aplicar nele o schema completo de produção (16 arquivos SQL, na ordem cronológica correta, excluindo dados reais da Clínica Hassum), habilitar `pg_cron`/`pg_net`, agendar o cron do `run-automations` apontando para o projeto novo, e fazer deploy das 8 Edge Functions com o `verify_jwt` correto.
- **Why:** É a fundação técnica de toda a Fase 2 — sem um banco isolado com schema e functions funcionando, não há onde construir nem revisar nada das Fases 2-5. A pesquisa (01-RESEARCH.md) confirmou que a sessão MCP disponível está estruturalmente isolada da organização de produção (credenciais diferentes), então isolamento aqui é garantido por autenticação, não apenas por disciplina.
- **Output:** Projeto Supabase `zapflow-fase2` ACTIVE_HEALTHY, com schema completo, extensões habilitadas, cron agendado e 8 edge functions no ar — pronto para receber env vars do Vercel (Plan 01-02) e dados/secrets de teste (Plan 01-03).
