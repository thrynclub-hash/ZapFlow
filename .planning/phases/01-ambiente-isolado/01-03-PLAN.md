# Phase 1 Plan 03: Secrets manuais, usuários Auth e seed de dados de teste

## Objective
- **What:** Coordenar as etapas que só podem ser feitas por Leonardo (secrets de Edge Function, credenciais de teste do Mercado Pago, criação de usuários reais de Supabase Auth) e, assim que esses pré-requisitos existirem, popular o banco isolado com dados de teste fabricados (contatos, campanhas, números, message logs) suficientes para a aplicação renderizar de ponta a ponta.
- **Why:** A pesquisa confirmou que nenhuma tool MCP disponível cria secrets de Edge Function nem usuários de Supabase Auth — são ações exclusivas de dashboard. Sem essas duas coisas, ninguém consegue logar na aplicação nem testar o fluxo de cobrança, mesmo com o schema e o deploy prontos (Plans 01-01/01-02).
- **Output:** Projeto `zapflow-fase2` com secrets de teste configurados, 2 usuários reais capazes de logar, e um conjunto mínimo de dados fabricados visível nas telas (Contacts, Campaigns, Reports, Birthdays).
